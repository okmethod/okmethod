# コーディングの心得

## 大原則：理解しやすいコードを書く！

- コードは「書かれる」よりも「読まれる」方が多い
- コードは他の人(未来の自分を含む) が最短時間で理解できるように書く
- コードを短く書くことは重要だが、理解にかかる時間を短くすることを優先する
- このコードは理解しやすいだろうか？と自問自答する

## [code: どのようにコードを書くか](./01-code.md)

コンセプト: 個々のコード片を読みやすく書く

トピック:

- 命名: 命名により、その目的や内容が一目でわかる程度に明確にする
- コメント: コメントにより、コードからは読み取れない意図や仕様を伝える
- コードスタイル: コーディングスタイルを統一し、読みやすく管理しやすい状態にする

## [logic: どのように機能を書くか](./02-logic.md)

コンセプト: 複雑な問題を、シンプルに解けるように書く

トピック:

- 変数とスコープ: 変数を扱いやすくし、コードの見通しを良くする
- 式と演算: 複雑な式を分割し、一目で理解できるようにする
- 制御フロー: 制御フローを単純にし、読み手が立ち止まったり読み返したりしなくて済むようにする
- 機能の分割と配置: 関数やクラスは、明確で単一の責任を持つように書く
- アルゴリズム選択: 問題を効率的に解くアルゴリズムやデータ構造を選択する

## [system : どのようにシステムを書くか](./03-system.md)

コンセプト: 現実世界で動かしやすい実用的なシステムとして書く

トピック:

- エラーハンドリング
- ロギング
- パフォーマンス
- セキュリティ
- テスト
