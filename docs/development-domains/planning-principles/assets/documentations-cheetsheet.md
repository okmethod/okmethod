# ドキュメンテーションチートシート

**_前提：システムの規模に応じて、記載粒度や書くこと・書かないことのバランスを見極めながら書く_**

## 業務要件定義書 構成例

```
タイトル 版数
変更履歴
目次
1. はじめに
2. 要求と達成目標
  2-1. ビジネス要求（Why）
  2-2. 達成すべき目標（What）
  2-3. 目標としない事項（Out of Scope）
3. 業務要件
  3-1. 業務の概要（コンセプトとビジネス価値）
  3-2. ドメイン用語の定義
  3-3. 登場人物の定義
    3-3-1. 組織モデル図（アクターと現実組織構造のマッピング）
    3-3-2. アクター一覧
  3-4. ユースケース
    3-4-1. ユースケース全体フロー
    3-4-2. ユースケース一覧（ユースケース・アクター・目標のマッピング）
    3-4-3. ユースケース詳細（各ユースケースの業務フロー等）
```

## システム要件定義書 構成例

```
タイトル 版数
変更履歴
目次
1. はじめに
2. 前提条件・制約条件
  2-1. 前提条件（前工程の合意内容、想定ユーザー、想定データ、等）
  2-2. 制約条件（技術的制約、法的制約、等）
  2-3. システム方式と境界設計
    2-3-1. 論理構成図
    2-3-2. 主要コンポーネントの説明（責務、I/Fの方針、処理方式、等）
3. 論理データモデル
  3-1. UMLクラス図（or 概念ER図）
  3-2. データモデル一覧（マスタデータ、トランザクションデータ、論理ファイル、等）
  3-3. データ量見積もり
4. インターフェース要件
  4-1. ユーザーインターフェース要件（画面要件）
    4-1-1. 画面構成図
    4-1-2. 画面一覧
    4-1-3. 画面仕様（各画面のワイヤーフレーム）
  4-2. システム間インターフェース要件（API要件・帳票要件）
    4-2-1. インターフェース一覧
    4-2-1. インターフェース仕様（プロトコル、データ形式、スキーマ、等）
5. 機能要件
  5-1. 機能配置図
  5-2. 機能一覧（機能・画面・ユースケースのマッピング）
  5-3. 機能仕様（各機能のIPO、計算ロジック、エラー処理方針、等）
6. 非機能要件
  6-1. SLA（サービスレベル合意）
  6-2. 性能目標
  6-3. 可用性
  6-4. 拡張性
  6-5. 可観測性
  6-6. データ保全
  6-7. 運用・メンテナンス
  6-8. セキュリティ
```

## 参考: Design Doc 構成例

```
タイトル
体制・問い合わせ先
1. はじめに
  1-1. システムorプロダクトの概要
  1-2. 背景・過去の経緯
2. 要求と要件（Requirements）
  5-1. 業務要件（or 業務要件定義書リンク）
  5-2. システム要件（orシステム要件定義書リンク）
3. 設計のコンセプト
  3-1. 設計の基本方針
  3-2. 将来の拡張の方針
4. システムアーキテクチャ設計
  4-1. システムをホストする環境の前提（物理機器選定 or クラウド選定）
  4-2. アーキテクチャ図（物理構成図）
  4-3. 各コンポーネントの説明（責務、技術スタック、等）
5. データ設計
  5-1. 物理データモデル（ER図、テーブル定義、インデックス設計、等）
  5-2. マイグレーション方針
  5-3. キャッシュ戦略
6. インターフェース設計（Contracts）
  6-1. HTTP API仕様（APIエンドポイント一覧、OpenAPI/Swagger定義へのリンク等）
  6-2. メッセージング・イベント駆動設計（Topic設計、Payload定義、等）
  6-3. 外部システム接続方式（認証、リトライ、タイムアウト、等）
7. 内部ロジック・アルゴリズム設計
  （コードをわかりやすく書く前提で、ここはドキュメントに書かない）
8. セキュリティ・コンプライアンス設計
  8-1. 認証・認可の実装方式（OAuth scope設計等）
  8-2. データ暗号化・保護方式（At rest / In transit）
9. テスト方針・運用設計
  7-1. 構築している環境
  7-2. テスト戦略（テスト工程の一覧と、検証範囲・環境のマッピング）
  7-3. 可視化・アラート設計（メトリクス、ログ、アラート通知、等）
10. 検討の記録
  10-1. 選定理由と代替案（技術選定・アーキテクチャパターン選定などの候補・比較検討・結論、等）
  10-2. 既知の制限事項・注意事項（できないこと、技術的リスクとコンチプラン、等）
付録・参考資料リンク
```
