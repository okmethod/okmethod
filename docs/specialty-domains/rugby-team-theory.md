## 選手団

### ポジション図

```js
--------FW--------
  [01]      [03]   # プロップ
       [02]        # フッカー
    [04]  [05]     # ロック
  [06]      [07]   # フランカー
       [08]        # ナンバーエイト
--------BK--------
       [09]        # スクラムハーフ
       [10]        # スタンドオフ
    [12]  [13]     # センター
 [11]        [14]  # ウィング
       [15]        # フルバック
```

### フォワード（FW）: 事業サイド

**フォワードの特徴**: 密集やゴールライン際のクリティカルな攻防

- 案件獲得、要件定義、契約面やPJスコープの泥臭い調整
- 顧客との接点で体を張るため、泥臭いパワフルさとパッションが求められる
- FW が弱いと、「ボール」（案件獲得・プロジェクト占有権）を確保できずチームが総崩れになり、負ける

#### **1,3番 プロップ**: シニアセールス / アカウントエグゼクティブ

- 契約締結や予算確保の際、顧客の無理難題（圧力）を正面から受け止める壁。
- 彼らが押し負けると、プロジェクトのスコープが際限なく広がり、BK（技術陣）まで圧迫される。

#### **2番 フッカー**: プリセールス / 営業リーダー

- スクラム（商談）の中心でボールを足に掛ける役割。
- 顧客の本音（ボール）を掻き出し、自社に有利な展開へと導く。

#### **4,5番 ロック**: データエンジニア / データサイエンティスト

- 膨大な生データを扱い、チームの背骨となる「事実」を積み上げ、固める。
- 積み上げたデータに基づいて高所から遠くまで見通すことで、商談と仮説のロジックの空中戦を制する。

#### **6,7番 フランカー**: セールスエンジニア / フルスタックエンジニア

- 攻守に顔を出し、ビジネスと技術の境界線で最も激しく動く。
- 顧客の要望を即座にプロトタイプに落とし込むなど、実装もサポートする機動力。

#### **8番　ナンバーエイト**: プロジェクトマネジャー / プロダクトマネジャー

- FW（事業）を統率しつつ、自らも突破口を開く。攻守の要となる中心人物。
- 事業価値を最大化するためにどこで勝負するか（「なぜ作るのか」「何を作れば勝てるのか」）を決める司令塔であり、予算・納期・品質を守り、円滑にプロジェクトを完遂させる現場責任者でもある。

### バックス（BK）: 技術サイド

**バックスの特徴**: キックやサインプレーで大幅ゲインする

- FW が体を張って供給したボールを、価値の源泉である「動作するソフトウェア」へとスマートに繋げる
- 安全なデリバリーとサービス提供のため、正確で精緻な技術と論理的思考力が求められる
- BK が弱いと、「ゲインとトライ」（プロジェクトの価値）を稼げず、勝てない

#### **9番 スクラムハーフ**: アーキテクト・テックリード

- 事業サイド（FW）と技術サイド（BK）を接続するハブ。
- 事業側が体を張って獲得した「混沌とした顧客要望（要求）」を要件として整理し、技術側が走りやすい「綺麗なパス（設計）」として供給する。
- FWの泥臭さと、BKの精緻さを繋ぐため、双方の言語を話せる必要がある。

#### **10番 スタンドオフ**: リードエンジニア

- チームの司令塔。状況を見て「次は攻めるか（実装）」「蹴るか（様子見/調整）」を判断する戦術決定者。
- 設計を具体的なタスク、WBS（サインプレー）に落とし込み、メンバーを動かして開発進捗（ゲイン）させる。
- 現場の実装をリードし、自らも華麗なステップで難所を実装（突破）する。

#### **12,13番 センター**: バックエンドエンジニア / インフラエンジニア

- 攻守の要。中核となる機能を形にする。
- 堅牢なロジックを組み、大量のトラフィックをさばき、UX の根幹を担う。

#### **11,14番 ウィング**: フロントエンド / デザイナー

- 最新技術やデザインで、一気にプロダクトの魅力を引き上げる仕上げ担当。
- 圧倒的なスピードと見栄えで、顧客が直接目にする価値（トライ）を届ける。

#### **15番 フルバック**: テストエンジニア / QA

- 最後の砦。最後方で全体を俯瞰して守備を固める。
- 堅牢なテスト計画・テスト設計で守備配置をコントロールしつつ、失点（バグ・リリース事故）を漏らさずシャットアウトする。

## スタッフ・その他

### コーチ / トレーナー / ヤカン: ミドルマネジャー

- 試合（プロジェクト）前からチームをサポートし、トレーニングやアドバイスを提供する。
- 試合中もフィールド外から全体を見守り、冷静に戦況を伝え、ピンチの時には駆け寄る存在。
- ここ一番の勝負所では激励の言葉と「魔法のヤカン」でチームを励ますが、「シンビン」や「怪我」が起きそうな場合はいち早くケアし、未然に防ぐ。

### 監督: 経営サイド

- 試合（プロジェクト）のフィールドには立たないが、勝つための「戦略（ビジョン）」と「予算（リソース）」を与える。
- 負けが込んだ時に責任を取るのも仕事。

### レフリー: 顧客の意思決定者

- どれだけ完璧なパス（設計）を回し、力強いスクラム（商談）を組んでも、レフリーが笛を吹けば試合は止まる。
- レフリーとコミュニケーションの「キャプテン」（チームの代表）の仕事であり、良好な関係を築く必要がある。

### 観客: 市場 / ユーザー

- 時に応援・熱狂し、時に厳しい野次を飛ばす、試合（システム・プロダクト）を評価する存在。
- 興業である以上、観客を喜ばせることは重要課題だが、決して観客に振り回されてはいけない。
- 厳しい野次（クレーム）よりも、空席（無関心）の方がずっと恐ろしい。

## ラグビー理論で見る ITプロジェクトあるある

### ペナルティあるある

- **ノットリリースザボール**  
  顧客情報や要望などの重要な情報（ボール）を共有できない状態。
  9番（テックリード）にパスが回らないため、BK（開発）がいつまでも走り出せず、停滞を招く。

- **オフサイド**  
  契約が巻けない。この状態で開発着手するとコンプラ的にまずく、反則となる。

- **ノットストレート**
  契約内容が提案・開発実態と乖離している。ちょうど真ん中でないとうまくいかない。

- **スローフォワード**  
  フィージビリティの低い無理な要件定義。
  FWからBKへのパスが、前方の「技術的に不可能な領域」へ放られること。
  ギリギリのラインと、反則のラインを見極めないと、プロジェクトが炎上する。

- **タッチキック**  
  攻めあぐねて自陣（納期直前）でピンチになった際、一度大きく外に蹴り出して、ラインアウト（再調整）からやり直すこと。
  納期調整やフェーズ分割を行い、体制を整える勇気。

- **ノックオン**  
  致命的なバグ。あってはならないことだが、常に発生するリスクがある。
  笛が鳴るまでではプレーオンなので、しっかりカバーに入らないとカウンターを食らって失点（信頼失墜・事故）につながるリスクもある。

- **シンビン**  
  キーマンの離脱。インフルエンザやトラブル対応で10分間（数週間）一時離脱すること。
  チームの危機的状況だが、残ったメンバーでなんとか持ち堪えることもできる。持ち堪えるしかない。

- **靭帯断裂**  
  致命的な怪我。オーバーワークやメンタル失調でチームから戦線離脱すること。
  チームの戦略が根底から崩れ、体制の見直しは避けられない。
  残ったメンバーだけでなんとかしろとなれば、怪我人は連鎖的に増え、チームは崩壊する。

### FW（営業） と BK（技術） あるある

- **FW(営業)は必ずノットテンメーターを獲りに行く**
  FW は、どんな状況でも条件反射で速攻で突っ込む。

- **FW（営業）が勝手にサインプレーを出す**
  BK に相談なく、営業が顧客と「できます！」と握ってきてしまうこと。
  ラインができていない状況でパス（無茶振り）を放られると、実装者が死ぬ。

- **BK（技術）はすぐキックで逃げる**
  FW が必死に案件を押し込もうとしているのに、BK が「技術的にリスクが高い」「工数が足りない」と、すぐにスコープ外へ弾き出そう（キック）すること。

- **BK(技術)はヘッドキャップを被らない**
  FW は大体ヘッドキャップ（スーツとネクタイ）を被っているが、
  BK は被っていたり被っていなかったりする。

## 15人制ラグビーと 7人制ラグビー

15人制ラグビーを **「規律と組織力で戦うウォーターフォール開発」** とすると、  
7人制ラグビーは **「少数精鋭の個人が越境と阿吽の呼吸で戦うアジャイル開発」** と言える。

### 7人制ラグビーのポジション図

```js
  [01]      [03]   # プロップ
       [02]        # フッカー
    [04]           # スクラムハーフ
       [05]        # スタンドオフ
          [06]     # センター
             [07]  # ウィング
```

### 7人制ラグビー（アジャイル開発）の特徴

- 圧倒的なスピード感と運動量
- 7人で、15人制と同じ広さのフィールドで戦う
- 選手 1人に要求される役割の広さ（マルチスキル）
- 全員がコンタクトできる（顧客と話せる）
- 全員が走れる（技術を理解しコードを書ける）
- 「魔法のヤカン」の重要性が爆上がりする
- セットプレー（スクラム・ラインアウト）が簡略化される
- 1人の 1つのミスが致命的な命取りになる

### 陥りがちな落とし穴

- 15人制（ウォータフォール）の常識で考えてしまう
  - ウォーターフォールの重厚長大な規律は、時に足を引っ張る場合もある
  - ベースを押さえつつも、アンラーニングも必要になる
- 7人制を「簡易ラグビー」と勘違いしてしまう（なんちゃってアジャイル）
  - 人数が少ないからといって簡単なものではなく、むしろ7人制の方がずっと高度
  - セットプレーが簡略化されるからといって、サインを決めずに適当にパスして走ればいい（ドキュメントを書かかずに、作りながら考えればよい）訳ではない
  - 阿吽の呼吸で瞬時に連携できるほどの圧倒的な練習量（シビアなコミュニケーションと合意形成・共通認識）が求められる
  - 未経験者には、それがわかりづらいということもある
