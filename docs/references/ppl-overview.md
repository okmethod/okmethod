# 個人情報保護法まとめ

## 個人情報の大分類

- **個人情報**: 「生存する個人」に関する情報で、氏名や生年月日などにより特定の個人を識別できるもの。
- **要配慮個人情報**: 個人情報のうち、特にセンシティブなもの。
- **個人データ**: 個人情報を含む情報で、検索可能な「個人情報データベース」として管理されているもの。
- **保有個人データ**: 個人データのうち、事業者が開示・訂正・利用停止等の権限を持つもの。

関係性としてはこのようになっており、サブセットになるほど管理義務が厳しくなる。
$$\text{要配慮個人情報} \subset \text{個人情報}$$
$$\text{保有個人データ} \subset \text{個人データ} \subset \text{個人情報}$$

## 「個人情報」の定義と留意事項

- **定義**: 特定の個人を識別できる情報、または個人識別符号が含まれるもの
  - 評価情報、公刊物等によって公にされている情報や、映像、音声による情報も含まれる。
  - 暗号化等によって秘匿化されているか否かを問わない。
  - 死者に関する情報は対象外となる。
    - ただし、死者に関する情報が、同時に遺族等の生存する個人に関する情報でもある場合は対象となる。
- **具体例**:
  - 氏名、住所、性別、生年月日、顔画像、等
  - ある個人の身体、財産、職種、肩書等の属性情報
  - 単体では識別できないが、他の情報と照合すれば容易に個人を特定できるもの
    - 顧客 ID、アカウント名、等
- **留意事項**
  - 利用目的の特定・明示義務
    - 利用目的外のデータ利用をシステム的に制限できるようにする。
  - 適正取得の義務
    - 取得時に必ず利用目的をユーザーに通知・同意を得る。

## 「要配慮個人情報」の例と留意事項

- **定義**: 個人情報のうち、不当な差別や偏見等の不利益が生じる恐れがあり、特に慎重な取り扱いを要するもの。
  - オプトアウト（本人が拒否しない限り提供可能とする仕組み）による第三者提供は認められていない。
  - 漏えい・滅失・毀損が発生した、または発生したおそれがある事態が生じた場合、速やかに個人情報保護委員会への報告および本人への通知を行う義務がある。
- **具体例**:
  - 人種、信条、社会的身分
  - 病歴、犯罪の経歴、犯罪被害の事実
  - 身体障害、知的障害、精神障害などの事実
  - 健康診断の結果（特定健診・特定保健指導の結果など）
- **留意事項**:
  - 取得の原則禁止
    - 要配慮個人情報の入力を求める要件を含めない。機能を実装しない。
  - 厳格な安全管理措置
    - 通常の個人データとは DB を分離し、セキュリティレベルを変える。
    - アクセスログを定期的に監査し、不正アクセスを速やかに検出できる仕組みを設ける。

## 「個人データ」の例と留意事項

- **定義**: 「個人情報データベース等」を構成する個人情報
  - 「個人情報データベース等」とは、体系的に構成され検索可能な状態の個人情報の集合体を指す。
  - 紙媒体であっても、一定の規則（五十音順、生年月日順、等）に従って整理・分類されており、検索可能な状態であれば個人情報データベースに当たる。
- **具体例**:
  - 顧客 DB に格納された顧客情報（氏名、住所、等）のレコード
  - ログサーバーに格納された IP アドレスと時刻の組み合わせ
  - メールアドレス管理台帳
  - 紙の会員登録用紙が、五十音順に並べ替えた上で挟まれているファイル
- **留意事項**:
  - 安全管理措置の義務
    - 通信時・保管時の暗号化を必須とする。
    - 個人データへのアクセスはロールベースアクセス制御で厳格に管理する。
  - 正確性の確保
    - 定期的なデータの棚卸棚卸し、変更履歴の追跡、および変更有無を確認する機能または業務フローを設ける。
  - 第三者提供の制限
    - 本人の同意なしに第三者へ提供する要件を含めない。機能を実装しない。
    - 提供する場合、提供先での利用目的や安全管理措置について監督する義務を負うため、業務・システム・契約上のリスクを確認する。

## 「保有個人データ」の例と留意事項

- **定義**: 個人データのうち、個人情報取扱事業者が、データ操作(※)を行うことのできる権限を有するもの
  - データ操作(※): 開示、内容の訂正、追加又は削除、利用の停止、消去及び第三者への提供の停止
  - 他社から委託を受けて一時的に預かっているデータは、委託元の企業が開示・削除の権限を持つため対象外となる。
- **具体例**:
  - 自社システム内に保管されているほとんどのデータ（委託を受けて保管しているデータを除く）
- **留意事項**:
  - 開示・訂正・利用停止の義務
    - 本人確認を行った上で、データの一括開示・削除・訂正を可能にする機能または業務フローを設ける。
  - 利用目的の通知義務
    - 本人から請求があった場合、利用目的を遅滞なく通知する機能または業務フローを設ける。
