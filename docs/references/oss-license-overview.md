# OSS ライセンスまとめ

## 「利用」と「頒布」

OSS に関わる義務は、「利用」か「頒布」かによって大きく異なる。

- **「利用」とは** : 組織内のサーバー上での実行、開発環境での利用、自社サービスとしての利用（SaaS 提供など）
- **「頒布」とは** : ソフトウェアのダウンロード販売、実行ファイルの提供、デバイスへの組み込み・販売

ライセンスによって、「頒布」にあたる条件に差異があるため注意。  
（例: AGPL は、ネットワーク経由でのサービス提供も頒布に当たる、等）

## 「利用」時の基本的な義務

- **保証の免責の理解** : OSS は無保証であり、欠陥があっても作者は責任を負わないことを理解し、利用者自身がリスクを引き受ける。
- **帰属表示義務** : 著作権表示とライセンス条文を、製品やドキュメントに含める。
- **改変の明示義務** : OSS のソースコードを改変した場合、オリジナルのコードと区別できるように変更点を明示する。

## 「頒布」時の追加の義務

- **ライセンス伝播の義務** : ライセンスの種類に応じて、派生成果物を同ライセンスで提供する。
  - **コピーレフト**
    - コピーレフトライセンスの OSS を含むソフトウェアを頒布する場合、そのソースコードを同ライセンスの OSS として公開することを義務付ける。
    - (代表例) `GPL`, `AGPL`, etc
  - **準コピーレフト**
    - "改変"されていない準コピーレフトライセンスの OSS を含むソフトウェアを頒布する場合、その OSS のオリジナルソースコードを入手できることを保証することを義務付ける。
    - "改変"された準コピーレフトライセンスの OSS を含むソフトウェアを頒布する場合、改変した部分のソースコードを公開することを義務付ける。
    - 成果物全体の公開義務は原則ないが、リンク方法によるため注意が必要。
    - (代表例) `MPL`, `LGPL`, etc
  - **非コピーレフト**
    - 上記のようなソースコードの公開義務はない。
    - (代表例) `MIT`, `BSD`, `Apache License 2.0`, etc
